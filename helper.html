This helper will assist you in creating JSON file(s), needed for translating your site , when using jQuery translator.

If you are using online translator like Google translate or similar.

  • Quick start

    Explore helper.html and helper.js and replace strings in var plain inside of function plain_translation ( language ) in helper.js with your own strings.

    Make sure the strings are in same order in each language array !!!

  • Adjust <options > </options > in <select id="to_language"> in helper.html to suit your needs.
    EXAMPLE :
    <option value = "en">english</option >
    <option value = "sk">slovak</option >
    <option value = "es">spanish</option >
    <option value = "pl">polish</option >

Create translation JSON file for your site.

Default to default + default to every other language!

Your JSON file will display here.

  • Step by step

    Explore helper.html and helper.js to get an idea, what you will be creating now!
  • Collect all the text you need translating.
  • Wrap it in double quotes "". EXAMPLE : "first part of the text", "second part of the text"
  • Create first key => value pair in var plain inside of function plain_translation ( language ) in helper.js
    EXAMPLE :

    var plain = {
    'en' : [
    "first part of the text",
    "second part of the text",
    "third part of the text",
    ]
    }
    See helper.js for example.
  • Take this text :

    "first part of the text",
    "second part of the text",
    "third part of the text",

    and translate it in Google translate or any other translator you use, and translate it to languages that you need, adding key => value pairs to var plain as with your default language as you go. Make sure that order of the strings stays the same, and that you don't loose any string in the process !

  • When using Google translate, regular double quotes " might get replaced with “ or „ . So just copy your text into text editor and replace them with correct ones " !
  • Adjust <options > </options > in <select id="to_language"> in helper.html to suit your needs.
    EXAMPLE :
    <option value = "en">english</option >
    <option value = "sk">slovak</option >
    <option value = "es">spanish</option >
    <option value = "pl">polish</option >
  • Create translation JSON for every language:

    default to default and save it in /assets/js/languages/${ default_language }.json

    If your default language is 'en' , then you would store it in
    /assets/js/languages/en.json

    and default to any other language and store it accordingly.

    EXAMPLE:
    /assets/js/languages/es.json
    /assets/js/languages/pl.json
    /assets/js/languages/sk.json
  • If you want to store your files in different directory, update
    path_to_language_files variable in settings.js
