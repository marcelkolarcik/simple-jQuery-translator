Usage

  • To use it in your project :

  • 1. Download assets/js/translator folder from simple Jquery translator repository and place it in the root directory of your project.

    Include assets/js/translator/translator.js in your HTML pages, that need translation.
  • <script src = "assets/js/translator/translator.js" type = "module" ></script >
  • If developing locally, you will need to run HTTP server. One of them can be found at local web server.
  • If you are using any of the online translators, feel free to download index.html, helper.html, usage.html, variables.html and assets/js/translator_helper directory and follow the instructions in helper.html to help you create translation JSON files faster.
  • Once you're done creating translation files using index.html, helper.html, usage.html, variables.html and assets/js/translator_helper files, you can remove those from your project, as they are not needed for production.
  • 2. Update assets/js/translator/settings.js
  • Open settings.js in assets/js/translator/settings.js and update variables :

    SITE'S DEFAULT LANGUAGE
    var default_language // Demo example : "en"

    PATH TO LANGUAGE FILES, UPDATE TO YOUR PROJECT'S FOLDER STRUCTURE
    var path_to_language_files // Demo example : "assets/js/languages"

    WHAT LANGUAGE FILES YOU ARE USING , FOR EXAMPLE:
    DEMO SITE IS USING en.json, sk.json, es.json, pl.json => site_languages = ["en","pl","sk","pl"].
    CASE'S OF LANGUAGES MUST MATCH LANGUAGE FILES!!!
    var site_languages // Demo example : ["en","pl","sk","pl"]

    FOR DEVELOPMENT ONLY IF YOU WANT TO HIGHLIGHT TEXT THAT IS NOT TRANSLATED TO ANY PARTICULAR LANGUAGE YET SET highlight_class THAT WILL BE APPLIED TO ELEMENT THAT IS NOT TRANSLATED YET TO HIGHLIGHT IT TO NOTIFY YOU!
    SWITCH OFF IN PRODUCTION
    var highlight_class = '' ;

    var highlight_class // Demo example : "bg-warning text-danger p-1"

    to match your site.

  • You can help yourself with helper.html and assets/js/translator_helper/helper.js
  • 4. Create your HTML elements that need translating using HTML Examples and Variables

  • First load of the page

    Translator will try and match any of site language(s) with browser language(s) by running translate(); function.

    If there will be match,page will be translated to matched language.

    Otherwise page will get translated to default language as set in
    assets/js/translator/settings.js

    In any case we will store this language into localStorage .

  • User changing language

    When user changes language, we will run translate(); function again and page will be translated to that language. And newly selected language will be set in localStorage.

    In the demo example, we are using bootstrap's dropdown, and 'click' event to trigger translate(); function.

    You can of course apply different logic for triggering translate(); function.
