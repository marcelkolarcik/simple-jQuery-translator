Usage

  • To use it in your project :

  • Download assets folder from simple Jquery translator repository and place it in the root directory of your project.
  • If developing locally, you will need to run HTTP server. One of them can be found at local web server.
  • If you are using any of the online translators, feel free to download index.html, helper.html, usage.html, variables.html and follow the instructions in helper.html to help you create translation JSON files faster.
  • Update settings.js
  • Open settings.js in assets/js/translator/settings.js and update variables :

    var default_language // Demo example : "en"
    var path_to_language_files // Demo example : "assets/js/languages"
    var site_languages // Demo example : ["en","pl","sk","pl"]
    var highlight_class // Demo example : "bg-warning text-danger p-1"

    to match your site.
  • Create translation JSON files and place them in assets/js/languages directory.
    You can help yourself with helper.html and assets/js/translator_helper/helper.js
  • Create your HTML elements that need translating using HTML Examples and Variables

  • First load of the page

    Translator will try and match any of site language(s) with browser language(s) by running translate(); function.

    If there will be match,page will be translated to matched language.

    Otherwise page will get translated to default language as set in
    assets/js/translator/settings.js

    In any case we will store this language into localStorage .

  • User changing language

    When user changes language, we will run translate(); function again and page will be translated to that language. And newly selected language will be set in localStorage.

    In the demo example, we are using bootstrap's dropdown, and 'click' event to trigger translate(); function.

    You can of course apply different logic for triggering translate(); function.
